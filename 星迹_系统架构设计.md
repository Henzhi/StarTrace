## 一、架构决策记录

### ADR-001: 离线优先架构策略

**状态**：Accepted

**上下文**：
产品的核心价值主张之一是"数据主权在用户"和"灵感捕捉不等待"。核心功能——碎片记录、星系浏览、故事生成（依赖本地可达LLM）——必须在完全无网络的环境下正常工作。同时，故事广场需要在线能力。

**决策**：
采用 **离线优先 (Offline-First)** 架构，Room 数据库作为唯一真实数据源 (Single Source of Truth)。具体策略：

```
┌─────────────────────────────────────────┐
│              UI Layer (Compose)          │
│         只观察 Room 数据 (Flow)           │
├─────────────────────────────────────────┤
│          Repository Layer                │
│   ┌──────────────┐  ┌────────────────┐  │
│   │ Local Source  │  │ Remote Source  │  │
│   │ (Room DAO)    │  │ (Retrofit API) │  │
│   └──────┬───────┘  └───────┬────────┘  │
│          │                  │           │
│   ┌──────┴──────────────────┴──────┐    │
│   │    NetworkBoundResource         │    │
│   │    Cache-Only 策略（本地优先）   │    │
│   └────────────────────────────────┘    │
└─────────────────────────────────────────┘
```

- **碎片/故事/LLM配置**：纯本地存储（Room），从不自动同步到服务器
- **故事广场数据**：NetworkBoundResource 模式（先展示缓存，后台刷最新）
- **用户登录态**：DataStore 存储 Token，本地 JWT 解析获取基本信息

**替代方案评估**：
| 方案 | 优势 | 劣势 | 判定 |
|------|------|------|------|
| 离线优先 (选择) | 核心功能永远可用；数据主权在用户；网络异常不影响体验 | 需处理数据一致性；云端同步延后到 v2 | ✅ 采用 |
| 在线优先 | 架构简单；无同步问题 | 核心场景（灵感捕捉）在网络差时无法使用，与产品定位冲突 | ❌ 拒绝 |
| 全离线 | 极简架构；零网络依赖 | 故事广场/用户系统不可行；失去社交价值 | ❌ 拒绝 |

**后果**：
- ✅ 碎片记录永远可用，满足"3秒内完成"的 P0 目标
- ✅ 用户数据隐私最大化（碎片和故事默认不上传）
- ⚠️ v1 缺少云端备份，须在 UI 中引导用户定期导出（风险缓解）
- ⚠️ v2 引入云同步时需要处理冲突解决（但 v1 先验证核心假设）

### ADR-002: 模块化策略 — Feature-First 分层解耦

**状态**：Accepted

**上下文**：
项目包含 4 个核心功能模块（星系、记录、故事、用户），每个模块有独立的数据模型、UI 和业务逻辑。需要一个既能保持模块边界清晰、又不引入过度抽象的模块化策略。

**决策**：
采用 **Feature-First + Clean Architecture 三层分离** 的混合策略：

```
app/
├── app/                        # Application, Hilt, DI 组装
├── core/                       # 全局基础设施（跨模块共享）
│   ├── data/                   # Room DB, DAO, Entity (共享)
│   ├── database/               # Database 定义 + Migration
│   ├── network/                # Retrofit, Interceptor
│   ├── security/               # Keystore, TokenManager
│   ├── di/                     # 全局 DI Module
│   └── util/                   # 纯函数工具类
├── feature/                    # 功能模块（每个模块自包含）
│   ├── galaxy/                 # 星系可视化
│   │   ├── data/               # ForceDirectedEngine, GalaxyRepository
│   │   ├── domain/             # LayoutAlgorithm, NodeInteractor
│   │   └── ui/                 # GalaxyCanvas, NodeDetailSheet, SearchBar
│   ├── fragment/               # 碎片记录
│   │   ├── data/               # FragmentRepository
│   │   ├── domain/             # FragmentInteractor, TagManager
│   │   └── ui/                 # RecordScreen, TagSelector, MoodPicker
│   ├── story/                  # 故事生成
│   │   ├── data/               # StoryRepository, LLMClient
│   │   ├── domain/             # StoryGenerator, PromptBuilder
│   │   └── ui/                 # StoryScreen, ConfigPanel, PolishChat
│   ├── auth/                   # 用户系统
│   │   ├── data/               # AuthRepository, TokenManager
│   │   ├── domain/             # AuthValidator
│   │   └── ui/                 # LoginScreen, RegisterScreen
│   └── profile/                # 我的页面 + 广场
│       ├── data/               # ProfileRepository, PublicStoryRepo
│       ├── domain/             # ProfileStats
│       └── ui/                 # ProfileScreen, PublicStoryList
└── design/                     # 设计系统（主题、组件）
    ├── theme/                  # Color, Typography, Shape
    └── component/              # StarButton, StarCard, StarToast...
```

**模块依赖规则**（编译时强制）：

```
                    ┌──────────┐
                    │   app/   │  ← DI 组装所有模块
                    └────┬─────┘
           ┌─────────────┼─────────────┐
    ┌──────┴──────┐     │     ┌───────┴──────┐
    │ feature/    │     │     │ feature/     │
    │ galaxy      │◄────┼────►│ fragment     │
    └──────┬──────┘     │     └───────┬──────┘
           │            │            │
           │      ┌─────┴─────┐      │
           └──────►  core/    ◄──────┘
                  └───────────┘

依赖规则：
1. feature/* 可以依赖 core/
2. feature/* 之间不直接依赖（通过 core/ 接口通信）
3. core/ 不依赖任何 feature/
4. app/ 可以依赖所有模块（用于 DI 组装）
```

**替代方案评估**：
| 方案 | 优势 | 劣势 | 判定 |
|------|------|------|------|
| Feature-First (选择) | 模块边界清晰；便于后续拆分为独立模块；单人开发时也不会混乱 | 初期文件结构稍复杂；需要额外的接口层避免循环依赖 | ✅ 采用 |
| Layer-First (controller/service/repo) | 传统、简单 | 功能跨越多个目录；模块边界模糊；后期拆分困难 | ❌ 拒绝 |
| 无模块（单 app 包） | 最简单 | 随功能增长不可避免成为大泥球 | ❌ 拒绝 |

**后果**：
- ✅ v2 云同步模块可作为新 feature/ 添加，不影响现有代码
- ✅ 每个 feature 可独立测试
- ⚠️ 需要纪律性维护模块边界（尤其单人开发时容易"抄近路"）
- ⚠️ 共享 Entity 在 core/data 中，feature 间可能通过 Entity 隐式耦合（接受此风险，因为核心数据模型相对稳定）

### ADR-003: 状态管理 — 单向数据流 (UDF) + StateFlow

**状态**：Accepted

**上下文**：
Android 开发中存在多种状态管理方案：LiveData、StateFlow、第三方库（如 MVI 库）。需要选择一个与 Compose 原生配合、类型安全、易于测试的方案。

**决策**：
采用 **Jetpack Compose 原生的单向数据流 (UDF) + Kotlin StateFlow**：

```
┌──────────────────────────────────────────┐
│              Composable UI                │
│  观察 StateFlow<UiState>                 │
│  通过 lambda 回调发射用户事件             │
├──────────────────────────────────────────┤
│              ViewModel                    │
│  private val _uiState = MutableStateFlow  │
│  val uiState: StateFlow<UiState>         │
│  fun onEvent(event: UiEvent)             │
├──────────────────────────────────────────┤
│         Domain Layer (UseCase)            │
│  纯业务逻辑，返回 Flow<Result<T>>         │
├──────────────────────────────────────────┤
│         Data Layer (Repository)           │
│  从 Room Flow 直接映射                   │
└──────────────────────────────────────────┘
```

**UiState 设计范式**（每个 Screen 一个 sealed interface）：

```kotlin
// GalaxyScreen 状态
sealed interface GalaxyUiState {
    data object Loading : GalaxyUiState
    data class Success(
        val nodes: List<GalaxyNode>,
        val selectedNode: GalaxyNode?,
        val isMultiSelectMode: Boolean,
        val filter: GalaxyFilter,
        val viewport: ViewportState
    ) : GalaxyUiState
    data class Empty(val message: String) : GalaxyUiState
}

// GalaxyScreen 事件
sealed interface GalaxyEvent {
    data class NodeClicked(val nodeId: String) : GalaxyEvent
    data class NodeLongPressed(val nodeId: String) : GalaxyEvent
    data class ViewportChanged(val newViewport: ViewportState) : GalaxyEvent
    data class FilterApplied(val filter: GalaxyFilter) : GalaxyEvent
    data object ResetView : GalaxyEvent
}
```

**命名约定**：
- `UiState` 总是 sealed interface，包含 Loading / Success / Empty / Error
- `UiEvent` 总是 sealed interface，每个事件一个子类
- ViewModel 不持有 `Context`、`View` 或任何 Android Framework 引用（通过 DI 注入需要的依赖）

**后果**：
- ✅ 状态变更可追溯（所有变更通过 Event → ViewModel → State 单向流动）
- ✅ 易于单元测试（ViewModel 是纯 Kotlin 对象）
- ✅ Compose 原生支持 StateFlow collectAsStateWithLifecycle()
- ⚠️ sealed interface 子类较多时需要更多样板代码（接受，类型安全优先）

### ADR-004: 星系渲染引擎 — Compose Canvas 自绘

**状态**：Accepted

**上下文**：
星系视图是产品的核心差异化交互，需要渲染数百个可拖拽的节点、贝塞尔曲线连线、深空粒子背景，并支持流畅的缩放和拖拽手势。

**决策**：
采用 **Compose Canvas API 自绘**，而非使用第三方图表库或 WebView：

```
GalaxyCanvas (Composable)
├── Canvas(modifier = Modifier.pointerInput(...))  // 手势层
│   ├── BackgroundLayer.draw()     // 深空背景 + 微尘粒子
│   ├── ConnectionLayer.draw()     // 半透明贝塞尔曲线
│   ├── NodeLayer.draw()           // 星点/星云节点
│   └── SelectionOverlay.draw()    // 选中高亮
├── GestureHandler                 // 单指拖拽/双指缩放/单击/长按
└── ForceDirectedEngine            // 布局算法（纯计算，不在 Canvas 内）
```

**数据处理与渲染分离**：

```kotlin
// 布局引擎：纯数据计算，结果缓存在 StateFlow
class ForceDirectedEngine @Inject constructor() {
    suspend fun calculateLayout(
        nodes: List<GalaxyNode>,
        iterations: Int = 50
    ): List<PositionedNode> {
        // 力导向迭代 → 产生 (x, y) 坐标
        // 无需 Canvas 上下文，完全可单元测试
    }
}

// 渲染：只做绘制
@Composable
fun GalaxyCanvas(
    nodes: StateFlow<List<PositionedNode>>,
    viewport: StateFlow<ViewportState>,
    onNodeClick: (String) -> Unit,
    modifier: Modifier = Modifier
)
```

**性能优化策略**：

| 节点数 | 策略 |
|--------|------|
| ≤ 50 | 全量渲染，每帧重绘所有节点 |
| 50-300 | 脏区域重绘 + 节点大小分级（越近的节点越精细） |
| > 300 | 自动切换到聚合视图（星座轮廓），详见产品设计 5.11 |
| 所有 | `drawWithCache` 缓存无关背景层；`graphicsLayer` 处理缩放平移变换 |

**替代方案评估**：
| 方案 | 优势 | 劣势 | 判定 |
|------|------|------|------|
| Compose Canvas 自绘 (选择) | 完全控制；与 Compose 手势系统集成；零额外依赖；可单元测试布局算法 | 开发成本高；需手动处理性能优化 | ✅ 采用 |
| 第三方图表库 (MPAndroidChart) | 快速上手 | 不支持力导向图；定制性差；不适合自由拖拽缩放 | ❌ 拒绝 |
| WebView + D3.js | 力导向图生态成熟 | 性能差（WebView 开销）；与原生手势割裂；离线场景需打包 JS | ❌ 拒绝 |
| OpenGL/Vulkan 自绘 | 极致性能 | 过度工程；开发成本极高；v1 节点数 < 300 不需要 | ❌ 拒绝 |

**后果**：
- ✅ 完全自定义的空间隐喻可视化，无第三方限制
- ✅ 布局算法可独立单元测试，不受 Canvas 影响
- ⚠️ 需要投入性能优化时间（Phase 4 专项优化）
- ⚠️ 团队 switch 时需要学习 Canvas API（但这是团队核心资产）

### ADR-005: LLM 集成 — 适配器抽象 + 拦截器链

**状态**：Accepted

**上下文**：
产品支持用户自配任意 OpenAI 兼容 API（OpenAI / DeepSeek / 通义千问 / 本地 Ollama），需要支持流式输出（SSE）和普通请求（连接测试）。

**决策**：
采用 **适配器模式 + OkHttp 拦截器链**：

```
┌──────────────────────────────────────┐
│          ViewModel                   │
│  storyGenerator(StoryConfig)         │
├──────────────────────────────────────┤
│          UseCase: GenerateStory      │
│  - 构建 SystemPrompt + UserPrompt   │
│  - 调用 LLMClient.stream()          │
├──────────────────────────────────────┤
│          LLMClient (接口)            │
│  interface LLMClient {               │
│    fun stream(config, request):      │
│      Flow<TokenEvent>               │
│    suspend fun test(config): Result  │
│  }                                   │
├──────────────────────────────────────┤
│   OpenAICompatibleClient (实现)       │
│  - OkHttp + SSE 解析                 │
│  - Interceptor Chain:                │
│    1. AuthInterceptor (API Key)      │
│    2. TimeoutInterceptor             │
│    3. RetryInterceptor (指数退避)     │
│    4. LoggingInterceptor (DEBUG)     │
├──────────────────────────────────────┤
│   LLMConfigRepository                │
│  - 从 Room 读取 LLMConfigEntity      │
│  - Keystore 解密 API Key             │
│  - 验证 URL 格式 + 连通性            │
└──────────────────────────────────────┘
```

**SSE 流式解析器**：

```kotlin
class SSEMessageParser {
    fun parse(line: String): TokenEvent = when {
        line.startsWith("data: [DONE]") -> TokenEvent.Complete
        line.startsWith("data: ") -> {
            val json = line.removePrefix("data: ")
            val token = extractContent(json)
            TokenEvent.Token(token)
        }
        else -> TokenEvent.Ignored
    }
}

sealed interface TokenEvent {
    data class Token(val text: String) : TokenEvent
    data class Error(val message: String, val retryable: Boolean) : TokenEvent
    data object Complete : TokenEvent
    data object Ignored : TokenEvent
}
```

**错误处理策略**：

| 错误类型 | HTTP 状态 | 用户感知 | 重试策略 |
|---------|----------|---------|---------|
| API Key 无效 | 401 | "API Key 验证失败" | 不重试 |
| 速率限制 | 429 | "请求过于频繁，请稍后再试" | 指数退避，最多 3 次 |
| 服务不可用 | 503 | "AI 服务暂时不可用" | 指数退避，最多 3 次 |
| 网络超时 | - | "连接超时，请检查网络" | 重试 1 次 |
| 内容被过滤 | 400 | "请求被服务端安全策略拦截" | 不重试 |

**后果**：
- ✅ 新增 LLM 供应商只需确认其 OpenAI 兼容性（绝大多数已兼容）
- ✅ SSE 解析器纯 Kotlin，不依赖 OkHttp SSE 插件（减少依赖）
- ✅ 通过接口抽象，可在测试中 Mock LLMClient
- ⚠️ 不同供应商的细微 API 差异需要单独适配（如通义千问的 `result_format: "message"` 参数）
- ⚠️ 流式输出中的错误恢复是难点（需保留已接收的 token，允许用户续写）

### ADR-006: 依赖注入 — Hilt 单模块 DI

**状态**：Accepted

**上下文**：
需要在整个应用中管理依赖关系——Room Database、Repository、UseCase、ViewModel——同时保持测试时能轻松替换实现。

**决策**：
采用 **Hilt (Dagger)** 进行编译时 DI，app 模块作为 DI 组装点：

```kotlin
// 全局 Module
@Module
@InstallIn(SingletonComponent::class)
object AppModule {
    @Provides @Singleton
    fun provideDatabase(@ApplicationContext ctx: Context): AppDatabase =
        Room.databaseBuilder(ctx, AppDatabase::class.java, "startrace.db").build()

    @Provides @Singleton
    fun provideFragmentDao(db: AppDatabase) = db.fragmentDao()

    @Provides @Singleton
    fun provideKeyStoreManager(@ApplicationContext ctx: Context): KeyStoreManager =
        KeyStoreManager(ctx)
}

// Feature-scoped Module
@Module
@InstallIn(ViewModelComponent::class)
object GalaxyModule {
    @Provides @ViewModelScoped
    fun provideForceDirectedEngine(): ForceDirectedEngine = ForceDirectedEngineImpl()

    @Provides @ViewModelScoped
    fun provideGalaxyRepository(fragmentDao: FragmentDao, storyDao: StoryDao): GalaxyRepository =
        GalaxyRepositoryImpl(fragmentDao, storyDao)
}

// ViewModel 通过 @HiltViewModel 自动注入
@HiltViewModel
class GalaxyViewModel @Inject constructor(
    private val galaxyRepository: GalaxyRepository,
    private val forceDirectedEngine: ForceDirectedEngine
) : ViewModel() { ... }
```

**Scope 设计**：

| Scope | 生命周期 | 适用 |
|-------|---------|------|
| Singleton | Application | Database, DAO, KeyStoreManager, Retrofit |
| ViewModelScoped | ViewModel | Repository, UseCase, ForceDirectedEngine |
| ActivityScoped | Activity | NavController（如果需要） |

**后果**：
- ✅ 编译时依赖图验证，不会出现运行时缺失依赖
- ✅ 测试时通过 @UninstallModules 轻松替换实现
- ⚠️ Hilt 增加构建时间（约 10-15%），但在本项目规模下可接受
- ⚠️ Hilt 学习曲线 a bit steep，但 Google 生态有大量文档

### 4.1 NetworkBoundResource 模式

对于需要在线数据的功能（故事广场），采用标准 `NetworkBoundResource` 模式：

```kotlin
// 通用封装
inline fun <ResultType, RequestType> networkBoundResource(
    crossinline query: () -> Flow<ResultType>,           // 本地缓存
    crossinline fetch: suspend () -> RequestType,         // 远程请求
    crossinline saveFetchResult: suspend (RequestType) -> Unit,  // 缓存远程数据
    crossinline shouldFetch: (ResultType) -> Boolean = { true }  // 是否更新
): Flow<Resource<ResultType>> = flow {
    emit(Resource.Loading())
    val data = query().first()
    val flow = if (shouldFetch(data)) {
        emit(Resource.Loading(data))  // 展示缓存，同时加载
        try {
            saveFetchResult(fetch())
            query().map { Resource.Success(it) }
        } catch (t: Throwable) {
            query().map { Resource.Error(t, it) }
        }
    } else {
        query().map { Resource.Success(it) }
    }
    emitAll(flow)
}

sealed class Resource<out T> {
    data class Success<T>(val data: T) : Resource<T>()
    data class Error<T>(val throwable: Throwable, val data: T?) : Resource<T>()
    data class Loading<T>(val data: T? = null) : Resource<T>()
}
```

## 七、星系渲染引擎设计

### 7.1 力导向布局算法

```kotlin
interface ForceDirectedEngine {
    suspend fun calculateLayout(
        nodes: List<GalaxyNode>,
        edges: List<GalaxyEdge>,
        config: LayoutConfig = LayoutConfig.Default
    ): List<PositionedNode>
}

data class LayoutConfig(
    val iterations: Int = 50,
    val attractionForce: Float = 0.01f,     // 标签相似度引力
    val repulsionForce: Float = 500f,       // 节点间斥力
    val centerForce: Float = 0.001f,        // 中心引力（防止飘散）
    val storyAttractionForce: Float = 0.05f, // 碎片→故事强引力
    val damping: Float = 0.9f               // 阻尼系数
)

class ForceDirectedEngineImpl @Inject constructor() : ForceDirectedEngine {
    override suspend fun calculateLayout(
        nodes: List<GalaxyNode>,
        edges: List<GalaxyEdge>,
        config: LayoutConfig
    ): List<PositionedNode> = withContext(Dispatchers.Default) {
        val positionedNodes = nodes.map { PositionedNode(it.id, it.positionX, it.positionY, it.type) }

        repeat(config.iterations) {
            // 1. 计算斥力（每对节点之间）
            for (i in positionedNodes.indices) {
                for (j in i + 1 until positionedNodes.size) {
                    val dx = positionedNodes[j].x - positionedNodes[i].x
                    val dy = positionedNodes[j].y - positionedNodes[i].y
                    val dist = max(sqrt(dx * dx + dy * dy), 1f)
                    val force = config.repulsionForce / (dist * dist)
                    val fx = force * dx / dist
                    val fy = force * dy / dist
                    positionedNodes[i].vx -= fx
                    positionedNodes[j].vx += fx
                    positionedNodes[i].vy -= fy
                    positionedNodes[j].vy += fy
                }
            }

            // 2. 计算引力（边连接 + 标签相似度）
            for (edge in edges) {
                val source = positionedNodes.find { it.id == edge.from } ?: continue
                val target = positionedNodes.find { it.id == edge.to } ?: continue
                val dx = target.x - source.x
                val dy = target.y - source.y
                val dist = max(sqrt(dx * dx + dy * dy), 1f)
                val force = if (edge.type == EdgeType.STORY_FRAGMENT) {
                    config.storyAttractionForce * dist
                } else {
                    config.attractionForce * dist
                }
                source.vx += force * dx / dist
                source.vy += force * dy / dist
                target.vx -= force * dx / dist * 0.1f  // 故事位置相对固定
                target.vy -= force * dy / dist * 0.1f
            }

            // 3. 中心引力
            for (node in positionedNodes) {
                node.vx -= node.x * config.centerForce
                node.vy -= node.y * config.centerForce
            }

            // 4. 应用速度 + 阻尼
            for (node in positionedNodes) {
                node.vx *= config.damping
                node.vy *= config.damping
                node.x += node.vx
                node.y += node.vy
            }
        }

        positionedNodes
    }
}
```

```kotlin
@Composable
fun GalaxyCanvas(

    edges: List<GalaxyEdge>,

                transformState = transformState.applyTransform(centroid, pan, zoom)

                onLongPress = { offset -> /* 长按检测 → onNodeLongPress */ }

        // 应用视口变换

            translate(transformState.offsetX, transformState.offsetY)

            // Layer 2: 节点
            drawNodes(positionedNodes, viewport)
        }

        // Layer 3: 叠加层（不随视口变换，固定在屏幕上）
        drawSelectionHighlight()
    }
```

### 9.1 API Key 存储

```kotlin
class KeyStoreManager @Inject constructor(
    @ApplicationContext private val context: Context
) {
    private val keyStore = KeyStore.getInstance("AndroidKeyStore").apply { load(null) }

    fun encrypt(keyAlias: String, plainText: String): ByteArray {
        val cipher = Cipher.getInstance("AES/GCM/NoPadding")
        cipher.init(Cipher.ENCRYPT_MODE, getOrCreateKey(keyAlias))
        return cipher.doFinal(plainText.toByteArray(Charsets.UTF_8))
    }

    fun decrypt(keyAlias: String, encrypted: ByteArray): String {
        val cipher = Cipher.getInstance("AES/GCM/NoPadding")
        cipher.init(Cipher.DECRYPT_MODE, getKey(keyAlias))
        return String(cipher.doFinal(encrypted), Charsets.UTF_8)
    }

    private fun getOrCreateKey(alias: String): SecretKey {
        val keyStore = KeyStore.getInstance("AndroidKeyStore").apply { load(null) }
        return if (keyStore.containsAlias(alias)) {
            (keyStore.getEntry(alias, null) as KeyStore.SecretKeyEntry).secretKey
        } else {
            val keyGenerator = KeyGenerator.getInstance(
                KeyProperties.KEY_ALGORITHM_AES, "AndroidKeyStore"
            )
            keyGenerator.init(
                KeyGenParameterSpec.Builder(
                    alias,
                    KeyProperties.PURPOSE_ENCRYPT or KeyProperties.PURPOSE_DECRYPT
                )
                    .setBlockModes(KeyProperties.BLOCK_MODE_GCM)
                    .setEncryptionPaddings(KeyProperties.ENCRYPTION_PADDING_NONE)
```

### 9.2 Token 管理

```kotlin
class TokenManager @Inject constructor(
    private val dataStore: DataStore<Preferences>,
    private val authApi: AuthApi
) {
    private val tokenFlow = dataStore.data.map { prefs ->
        prefs[ACCESS_TOKEN_KEY]?.let { JwtToken(it) }
    }

    fun getAccessToken(): Flow<JwtToken?> = tokenFlow

    suspend fun refreshToken(): Result<JwtToken> {
        val refreshToken = dataStore.data.first()[REFRESH_TOKEN_KEY]
            ?: return Result.failure(AuthException("No refresh token"))

        return try {
            val response = authApi.refreshToken(RefreshRequest(refreshToken))
            storeTokens(response.accessToken, response.refreshToken)
            Result.success(JwtToken(response.accessToken))
        } catch (e: Exception) {
            clearTokens()  // 刷新失败 → 清除 Token，要求重新登录
            Result.failure(e)
        }
    }

    suspend fun clearTokens() {
        dataStore.edit { prefs ->
            prefs.remove(ACCESS_TOKEN_KEY)
            prefs.remove(REFRESH_TOKEN_KEY)
        }
    }
}

data class JwtToken(val raw: String) {
    val isExpired: Boolean get() {
        val claims = parseClaims()
        return claims.expiration.time < System.currentTimeMillis() + 60_000 // 提前1分钟刷新
    }
}
```

```kotlin
class FeatureModuleBoundaryRule : Rule() {
    override fun visitImportDirective(importDirective: KtImportDirective) {
        val currentFile = importDirective.containingKtFile
        val importPath = importDirective.importedFqName?.asString() ?: return

        if (currentFile.packageFqName.asString().startsWith("com.startrace.feature.galaxy") &&
            importPath.startsWith("com.startrace.feature.fragment")) {
```
